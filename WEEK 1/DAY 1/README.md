# Week 1 – Day 1 AKRITI SRIVASTAVA

## System Reverse Engineering using Node.js

In this task, I created a Node.js script to understand basic system-level information and how Node.js can interact with the operating system.

### What I did
- Collected system details like hostname, disk space, open ports, default gateway, and logged-in users
- Logged CPU and resource usage using Node.js process APIs
- Stored the output in a JSON file inside a logs folder
- Created useful shell aliases in `.bashrc` to improve terminal productivity

### Files
- `sysinfo.js` – Node.js script for system metrics
- `logs/day1-sysmetrics.json` – Logged system output
